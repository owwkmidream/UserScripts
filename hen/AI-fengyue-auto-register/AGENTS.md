# AGENTS 协作规范

本文件是仓库内协作约束，适用于所有代码与文档改动。

## 任务前读取 INDEX

1. 在开始分析、设计、编码之前，优先读取根目录 `INDEX.md`，快速获取：
- 当前目录结构
- 入口与构建链路
- 模块职责与依赖边界

2. 如果发现 `INDEX.md` 与实际代码不一致：
- 以代码现状为准进行判断
- 在本次改动中同步修正 `INDEX.md`

## 强制规则：每次改动必须维护 INDEX.md

1. 任何改动（代码、配置、文档）都必须执行一次索引一致性检查。
2. 若改动影响索引语义，必须同步更新 `INDEX.md`。
3. 若改动不影响索引语义，可不改 `INDEX.md`，但提交说明必须写：
- `索引无变更（N/A）`

## INDEX 维护判定标准

出现以下任一情况，视为“影响索引语义”，必须更新 `INDEX.md`：

1. 文件结构变更
- 新增/删除/重命名文件或目录
- 关键文件移动位置

2. 入口或流程变更
- 运行入口变更（如 `src/index.js`、`src/app.js` 初始化链路）
- 构建链路变更（如 `rolldown.config.mjs` 输入/输出变化）

3. 模块职责变更
- 文件用途改变
- 依赖关系发生关键变化（例如核心服务层改为由其他入口直接驱动）

4. 外部边界变更
- API 基础地址、核心存储边界、GM 权限使用模式发生变化

## 提交前检查清单

- [ ] 已读取 `INDEX.md`
- [ ] 已核对本次改动是否影响索引语义
- [ ] 若有影响，已更新 `INDEX.md`
- [ ] 若无影响，提交说明已包含 `索引无变更（N/A）`

## 提交说明模板

```text
<type>: <summary>

Index: updated
```

或

```text
<type>: <summary>

Index: N/A（索引无变更）
```
